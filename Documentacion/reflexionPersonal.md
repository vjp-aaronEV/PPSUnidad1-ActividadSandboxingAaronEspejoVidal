# Reflexión personal
### Rust
En mi opinión uno de los lenguajes más seguros es **Rust**, ya que incorpora sistemas como el **borrow checker** entre otros, que comprueba automáticamente si el código tiene fallos que provoquen por ejemplo **stack overflow** o **null pointers** y da fallo antes de compilar.  
Es decir, Rust tiene gestión de memoria automática, pero sí que tiene la desventaja de que la curva de aprendizaje es muy alta.

---

### Java
Por otra parte, **Java** también tiene gestión automática de memoria, además de que el código Java se ejecuta en una **máquina virtual java (JVM)** que aísla hasta cierto punto la ejecución del código del sistema operativo.

---

### C / C++
En cambio también está C o C++ que aunque son lenguajes muy potentes gracias al control absoluto que tiene el programador sobre la memoria, esta misma ventaja es la que provoca que sean lenguajes propensos a fallos de seguridad por culpa del programados que no lo ha gestionado correctamente.

---

### Conclusión
En definitiva, aunque algunos lenguajes tienen sistemas de seguridad de manera automática que otros no, creo que lo que determina realmente si el lenguaje es seguro es que el programador utilice **buenas prácticas** del lenguaje y de programación, y que sea capaz de establecer las medidas de seguridad adecuadas.
